# /demo UI 시각 점검 결과(블루 톤 + 아우리봇 이미지)

## 점검 환경
- Base URL: `http://127.0.0.1:8787`
- Demo: `GET /demo`
- Static image: `GET /static/aouribot.png`

## 1) /demo 접속 가능 여부
- 결과: OK
- 확인: `GET http://127.0.0.1:8787/demo` → HTTP 200

## 2) docs/아우리봇.png 기반 이미지 노출 여부
- 결과: OK
- 확인:
  - `GET http://127.0.0.1:8787/static/aouribot.png` → HTTP 200
  - Content-Type: `image/png`

## 3) 시작 화면에 아우리봇 이미지가 보이는지
- 결과: OK
- 확인 포인트:
  - 히어로 카드에 `<img src="/static/aouribot.png">` 포함
  - 시작 문구:
    - “무슨 계약을 검토하고 싶으세요?”
    - “계약서를 붙여넣거나 업로드하면 …”

## 4) 결과 화면에도 이미지 또는 브랜딩 배지가 보이는지
- 결과: OK
- 확인 포인트:
  - 결과 패널 상단 배지에 아우리봇 소형 이미지 포함
  - 결과 상태 문구(`resultCopy`)가 “검토가 완료되었어요” 등으로 바뀌도록 구현됨

## 5) 전체 톤이 밝은 블루/하늘색 계열로 통일됐는지
- 결과: OK
- 확인 포인트:
  - `:root` 색상 변수 기반(white + light blue + point blue)
  - 배경 radial + 라이트 블루 그라데이션
  - 카드/버튼/탭/배지 스타일이 동일 계열로 통일

## 6) 버튼/카드/배지/UI 문구가 자연스러운지
- 결과: OK
- 확인 포인트:
  - 버튼: “질문 생성 / 검토 실행 / 수정 제안 보기 / 초안 작성 보기”
  - 카드: 검토 요약 / 배지(high risk/approval) / detected issues / 질문 답변 요약 / 수정 제안 / 초안 추천
  - 안내 문구가 딱딱하지 않고 “도와주는 법무 AI 비서” 톤으로 구성

## 7) 깨지는 이미지/깨지는 스타일/오타가 없는지
- 결과: OK(HTTP 기반 점검)
- 확인:
  - `/demo` HTML에 정적 이미지 경로가 포함되어 있고,
  - `/static/aouribot.png`가 실제로 200으로 서빙됨

## 브라우저에서 확인할 URL
- Demo: `http://127.0.0.1:8787/demo`
- Health: `http://127.0.0.1:8787/health`
- Static image: `http://127.0.0.1:8787/static/aouribot.png`

## 추천 테스트 방법(브라우저)
1) `http://127.0.0.1:8787/demo` 접속  
2) 시작 히어로 카드에 아우리봇 이미지가 보이는지 확인  
3) “검토 실행” 클릭 → 결과 카드/배지 업데이트 확인  
4) “수정 제안 보기” → “수정 제안 생성” 클릭  
5) “초안 작성 보기” → “추천” → “초안 생성” → “다운로드(.txt)”
