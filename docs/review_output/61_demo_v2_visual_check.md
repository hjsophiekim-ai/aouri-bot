# /demo(v2) 최종 점검(단일 질문 + 단일 결론)

## 접속 URL
- Demo(v2): `http://127.0.0.1:8787/demo`
- Demo(v1): `http://127.0.0.1:8787/demo-v1`
- Health: `http://127.0.0.1:8787/health`
- AouriBot image: `http://127.0.0.1:8787/static/aouribot.png`

## 점검 항목 결과
1) 시작 화면이 ChatGPT처럼 단순한지
- OK: 이미지 + 제목 + 최소 입력 필드 + “검토 시작”만 중심으로 노출

2) 계약서 첨부가 되는지
- OK: 시작 화면에 파일 첨부 input 존재, 업로드 시 `/api/upload`로 전송
- 참고(테스트 클라이언트): Windows PowerShell 환경 제약으로 multipart 수동 호출은 생략하고, 업로드 API는 테스트로 검증함
  - `python -m unittest -q runtime.tests.test_api_server` → OK

3) 법인/계약유형/기본 입력칸이 보이는지
- OK: 법인/계약유형/첨부/계약내용 입력 필드 존재

4) 질문이 한 번에 1개씩 나오는지
- OK: 진행 배지 `1/4` 형태 표시 + 질문 1개 출력 후 답변 입력 → 다음 질문

5) 결과가 하나의 대표 결론 중심인지
- OK: 대표 결론 1개 + 이유 요약(3~5줄) + 대표 추천 액션 1개

6) 수정 제안 확정 / 초안 작성 확정 버튼이 보이는지
- OK: 결과 화면 CTA 4개만 제공(확정 2개 + 다시 질문 + 처음)

7) 아우리봇 이미지가 첫 화면과 결과 화면에 보이는지
- OK: 시작(히어로) + 결과(상단) 이미지 노출

8) 전체 톤이 밝고 세련된 블루인지
- OK: white + light blue + point blue 계열의 단일 테마 유지

## 테스트 순서(1~5단계)
1) `http://127.0.0.1:8787/demo` 접속  
2) 법인/계약유형 입력 후, (A) 파일 첨부 또는 (B) 계약 텍스트 붙여넣기  
3) “검토 시작” 클릭 → 질문이 `1/4` 형식으로 1개씩 나오는지 확인  
4) 마지막 질문 답변 후 자동으로 결과 화면으로 이동하는지 확인  
5) 대표 결론/이유/대표 추천 액션/CTA(수정 제안 확정·초안 작성 확정) 확인 + 상세 보기 펼침 확인
