# MVP Gap Analysis (냉정 평가)

## 1) 지금 가능한 것
- 규칙 JSON 기반 키워드 탐지형 리뷰(LLM 없이)
- 질문/답변(룰 기반) 반영으로 적용 범위 확장
- 결과 저장/조회, 승인대기함 분리, EP 상태/결재 handoff 지점(stub/http)
- 조항별 수정 제안 뷰(진짜 redline 제외)


## 2) 지금 부족한 것
- 문맥 이해(예외/정의/범위/상호 참조) 부족
- 조항 파싱/정규화(번호/표/별첨/정의) 취약
- 오탐/미탐 통제(키워드 기반 한계)
- 표준 문구/대체 문안의 정교성 및 법인별 정책 세분화
- 성능/관측(로그/트레이싱/알람)과 운영도구 부족


## 3) 꼭 추가해야 할 기능
- 계약 유형/조항 분류 고도화(구조 기반)
- rule 트리거/스코프를 조항 단위로 적용(조항별 판정)
- rule 설명/근거(대표 문구, 근거 문서, 승인 정책) 강화
- 결과 검증을 위한 골든셋(라벨링) + 회귀 테스트 체계
- 사용자 수정/승인 워크플로우(사람 검토 기록/감사)


## 4) AI API가 들어가야 하는 기능
- 문맥 기반 의미 판정(면책 범위, 책임 제한, 간접손해, 고의/중과실 carve-out 등)
- 자동 redline/대체 조항 생성(표준 조항에 맞춰 리라이팅)
- 조항 요약/리스크 설명 자연어 생성(내부 교육/보고서용)
- 유사 판례/내부 기준 매칭(검색/리트리벌)


## 5) rule만으로 가능한 기능
- 필수 키워드/표현 탐지(무제한 책임/기술자료/대리점 비용 등)
- 체크리스트 생성(confirmed_standard 계열)
- 승인 필요/고위험 플래그로 워크플로우 라우팅
- 템플릿 기반 초안 생성(정형 계약)


## 6) 사람 검토가 반드시 필요한 기능
- 거래 구조/리스크 수용 여부 최종 판단
- 예외 승인(상대방 협상력/사업적 필요) 결정
- 사실관계 확인(개인정보 처리 범위, 하도급 여부, 현장 작업 여부 등)


## 7) EP 연동 전 준비해야 할 것
- EP 신청 데이터 표준화(필수 필드/첨부/버전)
- 결재 시스템 연동 요구사항(인증/멱등/재시도/콜백)
- 개인정보/기밀 데이터 처리 정책(로그 마스킹/보관기간)
- 운영 배포/모니터링/롤백 절차 확정


## 8) 우선순위별 다음 개발 단계
1) 룰 품질: 트리거 정교화 + 골든셋/회귀테스트
2) 조항 구조화: 조항 파서/타입 분류 + 조항별 판정
3) 워크플로우: 사람 검토 UI/히스토리/승인 정책
4) AI 도입: redline/대체문안/설명 생성(가드레일 포함)
5) EP/결재: 실연동(HTTP client 안정화, 콜백/폴링, 실패 복구)
