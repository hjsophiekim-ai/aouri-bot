# EP 연동 전 필수 준비 체크리스트 (MVP 기준)

## 목적
- 현재 AouriBot MVP를 EP에 붙이기 전에 “반드시 준비해야 하는 것”을 빠짐없이 확인한다.
- “지금 가능한 것 / 아직 안 되는 것”을 명확히 분리한다.

## 1) 기술 준비
- 런타임 실행/헬스체크
  - `/health`가 항상 OK인지
  - 서버 재시작 시 rules 로드/DB 마이그레이션이 안전하게 수행되는지
- API 계약(Contract) 고정
  - `/api/review/analyze` 응답에서 `summary.high_risk`, `summary.approval_required` 사용 방식 확정
  - 질문/수정제안/초안 API의 입력/출력 필드 명칭 확정
- 에러/타임아웃 정책
  - 입력 텍스트가 비어있거나 너무 짧을 때의 동작(에러/빈 결과) 확정
  - 긴 계약서 입력 시(대용량) 제한/분할 정책 확정
- 데이터 저장 정책
  - EP 연동 시 원문 저장 여부(저장한다면 접근권한/보관기간/마스킹 필수)
  - request_id/session_id와 EP 신청 ID의 매핑 규칙 확정

## 2) AI API 준비(향후)
- provider/config 구조는 준비됨(키 없이 mock 동작)
- 준비 항목
  - 어떤 기능부터 AI를 붙일지 우선순위 확정(설명 생성/수정문안/Q&A 등)
  - 모델/비용 한도/토큰 상한/캐시 정책 정의
  - 프롬프트/출력 스키마(구조적 JSON) 표준화
- 보안
  - API 키 저장 위치(ENV/secret manager)와 접근 권한 확정
  - 로그에 원문/키가 남지 않도록 정책 확정

## 3) 보안 준비
- 접근 통제
  - EP 사용자/법무팀/운영자 권한 구분(조회/수정/결재 전환)
  - Admin UI 노출 범위(사내망 제한 또는 인증 필요 여부)
- 로그/감사
  - 요청/결과/상태전이/결재 handoff에 대한 감사 로그(누가/언제/무엇을)
  - 민감정보 마스킹 규칙(계약서 원문, 개인정보, 계좌, 연락처 등)
- 데이터 거버넌스
  - 보관기간/삭제 정책
  - 외부 AI 전송 범위(필요 조항만 전송 등)

## 4) 테스트 준비
- 회귀 테스트
  - “골든셋” 샘플 계약 텍스트 + 기대 결과(라벨) 구축
  - 핵심 위험 문구 6종(무제한책임/면책/기술자료/단가감액/비용전가/안전) 회귀 테스트 유지
- 성능 테스트(최소)
  - 대표 길이(짧음/보통/긴 계약) 응답 시간 측정 기준 마련
- 장애 테스트
  - rules 파일 깨짐/누락 시 동작(서버 시작 실패 vs 제한 모드) 정책 확정
  - DB 잠금/파일 권한 문제 시 복구 절차 확정

## 5) 법무팀 검증 준비
- 파일럿 평가 키트 공유 및 평가 기준 합의
  - “법률 판단 자동화 금지” 가이드 포함
- 결과 해석 가이드
  - high_risk/approval_required 의미(업무 라우팅 신호)와 오남용 방지 문구
- 개선 피드백 루프
  - 미탐/오탐 보고 양식 + 우선순위 기준(P0/P1/P2)

## 6) EP 연동 시 필요한 IT 정보
- EP에서 제공 가능한 식별자/필드 정의
  - ep_request_id, 작성자/부서, 계약유형/거래유형, 상대방, 목적, 금액, 기간, 첨부, 상태 값
- 인증/네트워크
  - EP → AouriBot 호출 인증 방식(사내 SSO/JWT/내부망)
  - AouriBot API 접근 허용 IP/도메인/방화벽 정책
- 첨부 처리
  - 업로드 파일의 저장 위치(EP 스토리지 vs AouriBot 스토리지)
  - 텍스트 추출 포맷 확대 계획(PDF/OCR/HWP 등은 후속)

## 7) 결재 연동 전제조건
- 결재 시스템 연동 방식 확정
  - 실제 외부 API 존재 여부 / 인증 / payload / callback 여부
  - 멱등키/재시도/실패 복구(현재 MVP는 stub/http 지점만 제공)
- 전환 조건 합의
  - high_risk 또는 approval_required → approval_pending 전환
  - low/medium은 legal_review_pending 후 결재 전환 허용 정책

## 8) 지금 가능한 것 / 아직 안 되는 것
### 지금 가능한 것(GO for demo)
- `/demo` 단일 화면으로 EP 없이 시연(질문/검토/수정제안/초안)
- EP Mock 기반으로 상태 전이/결재 handoff 지점 시연
- rule 버전 조회/필터/고위험 조회/백로그 분리(Admin)

### 아직 안 되는 것(HOLD)
- 실제 결재 시스템 “실연동” 완성(현재는 연결 지점/저장/복구만 MVP)
- 문맥 기반 법률 판단(키워드 기반 한계)
- 자동 redline 생성/자동 반영
- OCR/PDF/HWP 등 비지원 포맷의 안정적 텍스트 추출
