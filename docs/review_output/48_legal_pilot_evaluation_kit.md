# 법무팀 파일럿 평가 키트 (MVP 1차 평가용)

## 목적
- AouriBot MVP를 “실제 법률검토에 쓸 수 있는 정도”와 “실무 투입 가능”을 분리해 냉정하게 평가한다.
- 이번 파일럿은 **의사결정 자동화**가 아니라, **리스크 후보 탐지 + 설명가능성 + 워크플로우 적합성**을 평가한다.

## 1) 평가 대상 기능
### A. Rule 기반 검토
- `/api/review/analyze` 결과(이슈 탐지, 적용 rule, 요약 플래그 high_risk/approval_required)
- backlog rule이 판정에 섞이지 않는지

### B. 질문 엔진
- 사전 질문 생성(`/api/questions/generate`)
- 답변 반영 시 검토 결과 변화(derived_context, matched_rules 변화)

### C. 수정 제안 뷰
- 조항별 이슈 표시(`/api/revision/suggest_text` 또는 세션 기반 `/api/revision/suggest`)
- 추천 수정 방향/대체 문안 품질(설명가능성)

### D. 템플릿 초안 작성
- 템플릿 목록/추천/초안 생성/다운로드(`/api/draft/*`)

### E. 운영/조회
- Admin 화면(`/admin`)에서 rule version/필터/고위험 조회/백로그 분리 확인

## 2) 테스트 시나리오(권장 6개)
각 시나리오별로 “계약 텍스트/파일” 1건씩 준비(총 6건).

1) NDA(비밀유지)
- 목적: 표준 체크리스트/예외 룰 동작 확인

2) 물품공급/구매
- 포함: 손해배상 상한 부재, 일방 면책, 기술자료 요구, 해지 불균형

3) 대리점/유통
- 포함: 판촉비/반품비/광고비 비용전가, 목표 미달 패널티 등

4) 용역/자문
- 포함: 산출물 귀속, 검수/대금지급, 책임 제한

5) 개인정보 처리(처리위탁/DPA)
- 포함: 위탁 범위/재위탁/파기/보안/국외이전 등

6) 현장/설치/안전 이슈
- 포함: 안전책임 일방전가, 산업안전/중대재해 관련 의무

## 3) 평가 항목(체크리스트)
### 정확도
- 미탐(놓침): 중요한 위험 문구가 있는데 탐지되지 않는 비율
- 오탐(불필요): 위험이 아닌데 위험으로 분류되는 비율
- 계약유형 입력이 흔들릴 때도 핵심 위험이 최소한 탐지되는지

### 유용성
- 법무 검토 시간을 줄이는 데 도움이 되는지
- “어디를 먼저 볼지” 우선순위가 합리적인지(요약 플래그/정렬)

### 설명가능성
- 어떤 rule이 적용되었는지 명확한지(rule_id/title/risk_level/keywords)
- 수정 방향/대체 문안이 실제 업무에 도움이 되는지

### 속도/안정성
- 응답 속도 체감(대략적인 범주: 빠름/보통/느림)
- 에러 없이 끝까지 흐름이 돌아가는지

### 워크플로우 적합성
- approval_required/high_risk 기반 라우팅이 실무 정책과 맞는지
- “사람 검토가 필요한 지점”이 UI에서 분명히 보이는지

## 4) 평가 기준(점수화)
각 항목 1~5점(1 매우 부족, 3 보통, 5 매우 우수).

### A. 정확도(중요도 최상)
- 5: 주요 위험 문구 미탐 거의 없음(파일럿 범위 내), 오탐이 업무를 방해하지 않음
- 3: 키워드 기반 한계는 있으나 “1차 스크리닝”으로는 실용
- 1: 미탐/오탐이 많아 참고자료로도 위험

### B. 유용성
- 5: 법무가 바로 체크/코멘트에 활용 가능
- 3: 참고자료로는 유용, 추가 설명이 필요
- 1: 표시가 복잡하거나 행동으로 연결되지 않음

### C. 설명가능성
- 5: 왜 이슈인지/어떻게 수정할지 이해 가능
- 3: rule/키워드는 보이나 설명이 부족
- 1: 결과가 “검은상자”처럼 보임

### D. 속도/안정성
- 5: 오류 없이 안정적으로 반복 실행 가능
- 3: 가끔 실패/느림 있으나 원인 추적 가능
- 1: 자주 실패하거나 복구 어려움

### 개선 요청
- 요청 요약:
- 현재 불편한 점:
- 원하는 UI/출력 형태:
- 기대 효과(시간 절감/오류 감소 등):
- 우선순위:
  - 높음 / 보통 / 낮음

## 7) go/no-go 판단 기준
### GO (다음 단계 진행)
- P0 없음
- 핵심 시나리오 6개 중 5개 이상에서 “정확도” 3점 이상
- Admin에서 rule 버전/필터/고위험 조회가 가능하고, 결과 근거가 추적 가능

### HOLD (보완 후 재평가)
- P1 버그 존재(특정 계약유형에서 반복 미탐/오탐)
- 안전/대리점 비용전가 등 핵심 리스크에서 탐지 일관성이 부족

### NO-GO (현 단계 제한)
- P0 존재 또는 “정확도” 평균 2점 이하
- 결과가 결정 자동화에 오남용될 위험이 높고, 통제 장치(권한/감사/가이드)가 없음
