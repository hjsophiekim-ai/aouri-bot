# /demo UI 문구 업데이트(시작/결과)

## 시작 화면(요구사항 반영)
### 큰 제목
- “무슨 계약을 검토하고 싶으세요?”

### 부제
- “계약서를 붙여넣거나 업로드하면 아우리봇이 먼저 검토 포인트를 찾아드릴게요.”

### 입력 영역 구성
- “계약 정보” 카드 내에 자연스럽게 배치
  - 법인(entity)
  - 계약유형(contract_type)
  - 계약 내용(텍스트)

### 이미지 배치
- 시작 히어로 카드 우측에 아우리봇 이미지(`/static/aouribot.png`)를 배치해 친근한 첫 인상을 제공

## 결과 화면(요구사항 반영)
### 결과 상단 문구/브랜딩
- 결과 카드 상단 우측에 아우리봇 배지(소형) 표시
- 상태 문구(`resultCopy`)를 상황별로 업데이트:
  - 기본: “아직 검토 전이에요”
  - 질문 생성 후: “질문을 준비했어요”
  - 검토 실행 후: “검토가 완료되었어요”
  - 수정 제안 생성 후: “수정 제안을 정리했어요”
  - 초안 추천 후: “초안 템플릿을 추천했어요”

### 요약/배지/카드 문구
- 검토 요약 카드: “검토 요약”
- 배지 카드: “high risk / approval required”
  - HIGH RISK 있음/없음
  - APPROVAL REQUIRED/결재 필요 없음
- detected issues 카드: “detected issues”
  - 이슈가 없으면 “검출된 이슈가 없어요. (체크리스트를 확인해 주세요)”
- 질문 답변 카드: “질문 답변 요약”
  - 답변이 없으면 “선택한 답변이 아직 없어요.”
- 수정 제안/초안 추천 카드:
  - “수정 제안”, “초안 추천”과 함께 “열기” 버튼 제공
