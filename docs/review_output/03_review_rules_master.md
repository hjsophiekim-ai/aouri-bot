# 03. 아우리봇 검토 기준 통합문서 (Master)

## 1. 문서 목적
- 본 문서는 아우리봇의 계약 검토 운영 기준을 통합한 실행 문서다.
- 본 버전은 `CONFIRMED` 근거만으로 확정 rule을 구성한다.
- `ESTIMATED / FAILED / OCR 필요` 항목은 확정 rule에서 제외하고 backlog로 분리한다.
- 적용 범위: `docs/Contract`, `docs/Standard Contract` 기반 검토 업무.
- 근거 문서:
  - `docs/review_output/01_standard_contract_summary.md`
  - `docs/review_output/02_contract_common_patterns_verified.md`
  - `docs/review_output/02a_contract_text_extraction_status.md`
  - `docs/아우리봇 법인별 법률검토 기준문서_v1.docx` (추출본: `docs/review_output/extracted_아우리봇_법인별_법률검토_기준문서_v1.txt`)

## 2. 상위 원칙
- 원칙 P1 (`표준 확정`): 확정 rule은 `CONFIRMED` 근거(본문 확인)만 사용.
- 원칙 P2 (`반복 패턴`): 빈도는 파일 기준으로 산정하며 `HIGH/MEDIUM/LOW`로 관리.
- 원칙 P3 (`예외 가능`): 법인 특성, 거래구조, 강행법규 충돌 여부에 따라 예외 검토 가능.
- 원칙 P4 (`추가 승인 필요`): 무제한 책임, 일방 면책, 안전책임 공백, 하도급 단가감액, 대리점 비용전가, 기술자료 요구는 법무 승인 필수.
- 원칙 P5 (`미확인`): `ESTIMATED/FAILED/OCR 필요`는 확정 판단 금지, backlog로만 유지.
- 원칙 P6 (`구조화`): 모든 rule은 `Rule ID / 상태 / 트리거 / 액션 / 근거` 필드를 유지.

## 9. 예외 가능 / 추가 승인 필요 사항
- 예외 가능
  - 안전책임 공백 후보(`RISK-003`): 부재기반 탐지라 계약유형별 재확인 필요
  - 독점/경업(`C-016`): 사업전략상 필요 시 범위·기간·지역을 제한해 승인 가능
  - 라이선스/로열티, 합의/정산/해지 유형: 건수 적음, 케이스별 법무 검토 필요
- 추가 승인 필요(법무/승인권자)
  - 무제한 책임, 일방 면책/배상, 기술자료 요구, 하도급 단가감액, 대리점 비용전가
  - 상장사 내부통제·공시·평판 리스크 연동 사안(퍼시스/시디즈/홀딩스 우선)
