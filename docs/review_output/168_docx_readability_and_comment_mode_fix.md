# 168. Word 가독성/코멘트 모드 개선(필수 redline + 권장/참고 guidance 분리)

## 문제
- redline은 동작하지만 “수정이 크게 잡혀” 가독성이 떨어지고,
- 부록의 수정 포인트가 토큰 나열(예: `계약, 종료, 시, 상대방는`)처럼 부자연스럽다.
- 모든 조항이 빨간 수정 대상으로 보이는 인상이 강하다.

## 목표
1) 본문 redline은 수정된 조항만 표시하되 조항 위치(조/항/호)와 맥락이 명확해야 함  
2) 부록의 change point를 자연어로 개선(토큰 나열 금지)  
3) 부록 컬럼 개선: 조항 위치 / 수정 전 핵심 표현 / 수정 후 핵심 표현 / 수정 이유 / 관련 법령/기준  
4) 고위험(HIGH)만 본문 redline  
5) 중간/중간이하(MEDIUM/LOW)는 파란색 guidance 섹션 분리  
6) Word 문서에 수정 수준(필수수정/권장수정/참고제안) 표시  

## 변경 사항
### 1) 조항 위치 표시 강화(조/항/호/목)
- `display_path`를 Word/앱 모두 우선 표시하도록 개선(165 연동)

### 2) risk tier 기반 문서 구조
- 파일: [docx_writer.py](file:///c:/Users/FURSYS/Desktop/aouribot/aouri-bot/runtime/review/docx_writer.py)
- 요약 섹션에 카운트 추가:
  - `필수수정(HIGH) / 권장수정(MEDIUM) / 참고제안(LOW)`
- 본문 redline은 `HIGH`만 표시:
  - 헤딩에 `[HIGH] 제n조 제m항 k호 ...` 표시
  - 하위 조항은 상위 맥락(`context_text`)을 회색으로 함께 표기
- `MEDIUM/LOW`는 “권장/참고(guidance)” 섹션으로 분리:
  - 방향(suggested_direction)
  - 사유(rewrite_reason)

### 3) 부록 표 컬럼/내용 개선
- 기존: 원문 요지 + 토큰 기반 “추가/강화” 나열
- 변경:
  - 조항 위치
  - 수정 전 핵심 표현(삭제된 핵심 구간)
  - 수정 후 핵심 표현(추가된 핵심 구간)
  - 수정 이유(자연어)
  - 관련 법령/기준
- “수정 전/후 핵심 표현”은 redline diff를 기반으로 추출하여 토큰 나열을 방지한다.

## 기대 효과
- Word에서 “필수 수정(빨강)”과 “권장/참고(파랑)”가 분리되어 과잉 redline 인상이 감소한다.
- 부록이 토큰 나열이 아니라 자연어 중심으로 읽히며, 수정 전/후 핵심이 명확해진다.
