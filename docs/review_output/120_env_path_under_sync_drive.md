# 동기화 경로(구글드라이브) 환경에서의 .env 경로 점검(120)

## 관측/리스크
- 프로젝트가 동기화된 바탕화면/드라이브 경로에서 실행될 때, 실행 cwd가 프로젝트 루트와 달라 `.env` 탐색이 꼬일 수 있다.

## 조치
- 디버그 엔드포인트를 통해 아래를 즉시 확인 가능하게 했다(값 미노출).
  - 실제 서버 cwd
  - repo_root(서버 코드 기준)
  - `.env` 후보/실제 로드된 경로
  - 프로젝트 루트 `.env`와 `aouri-bot/.env` 존재 여부

## 확인 방법(실행 중 서버 기준)
- `GET /api/debug/env-status`
  - `cwd`, `repo_root`, `dotenv.loaded`, `dotenv_exists`를 확인하면 경로 꼬임을 바로 진단할 수 있다.

## 다음 단계
- 경로가 꼬여도 안정적으로 동작하도록 `.env` 로딩 우선순위를 `aouri-bot/.env` 최우선으로 고정(121)했다.
