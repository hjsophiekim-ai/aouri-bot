# EP 화면 내 AouriBot 패널 UX(MVP)

## 목표
- “복잡한 기능”보다 **실제 써볼 수 있는 MVP UX**를 우선한다.
- EP 본 화면 흐름을 해치지 않도록 **우측 패널(탭 구조)**로 AouriBot을 제공한다.
- 한국어 UI를 기본으로 한다.

## 필수 UI 요소(구현 반영)
EP Mock 화면에서 아래 요소를 모두 포함한다.

### 1) 대화창
- 목적: 챗봇처럼 “진행 중이라는 느낌”과 기록(메모)을 제공
- 구현: `chatLog` 영역(봇/사용자 버블)
- 주의: MVP에서는 LLM이 없으므로, 채팅 입력은 “기록용”이고 실제 답변은 질문 폼에서 선택

### 2) 질문/답변 입력
- 질문은 rule 기반으로 생성되고, 각 질문은 select로 답변한다.
- 구현: `questionArea`에 질문 박스 렌더링
  - “답변 저장 & 검토 실행” 버튼으로 저장 + analyze 실행

### 3) 추천 질문 버튼
- 사용자가 바로 클릭해 “자주 묻는 질문”을 확인할 수 있게 한다.
- 구현: quick buttons(개인정보/현장작업/대리점/기술자료)

### 4) 현재 검토 상태 표시
- EP 신청 상태를 배지로 표시하고, 필요 시 수동 변경도 가능하게 한다(데모/운영 편의).
- 구현:
  - `statusBadge` + `statusSelect` + “상태 변경”
  - 상태 조회/전이 검증 API 연동

### 5) detected issues 요약
- “검토 결과가 나왔다”는 핵심 요약을 패널 상단에 배치한다.
- 구현:
  - `issueBadge` (matched rule count 기반)
  - `approvalBadge` (approval_required count 기반)
  - 상세는 결과 JSON(`reviewOut`)에서 확인

### 6) 초안 작성 탭
- 표준계약서 템플릿 기반 초안 생성(LLM 없이)
- 구현: “계약서 초안” 탭

### 7) 수정 제안 탭
- 진짜 redline 이전 단계의 “수정 제안 뷰”
- 구현: “수정 제안” 탭

### 8) 결재로 넘기기(다음 단계) 버튼
- 아우리봇 검토 완료 후, 다음 단계로 전환한다.
- 구현:
  - “다음단계: 자동 / 결재(법무확인 완료)” 선택 + “다음 단계로 넘기기” 버튼
  - 자동: high_risk/approval_required면 결재 전환, 아니면 법무 확인 단계로 라우팅
  - 법무 확인 후 결재: `force_approval=true`

## 화면 구조 추천(우측 패널)
MVP 기준 권장 구조는 다음과 같다.
- 좌측: EP 신청 입력 폼(기존 흐름 유지)
- 우측(고정/스티키): AouriBot 패널
  - 탭: 검토 / 초안 / 수정제안
  - 상단: 상태/요약(배지)
  - 본문: 대화창 + 질문/답변 폼 + 결과

Mock 구현: [ep_legal_request_ui.py](file:///c:/Users/FURSYS/Desktop/aouribot/aouri-bot/runtime/admin/ep_legal_request_ui.py)

## MVP 사용자 흐름(패널 기준)
1. EP 폼 입력 + 계약서 파일 선택
2. “아우리봇 패널 열기”
3. “검토 시작” → 질문 생성
4. 질문 답변 선택 → “답변 저장 & 검토 실행”
5. 결과 확인(요약 배지/JSON)
6. “계약서 초안” 또는 “수정 제안” 탭 확인
7. “다음 단계로 넘기기”
   - 자동이면 법무 또는 결재로 라우팅
   - 법무 확인 이후에는 “결재(법무확인 완료)”로 강제 전환 가능
