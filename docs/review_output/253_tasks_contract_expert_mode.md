# 253. Tasks: Contract Expert 모드 적용 작업

## 목표
- 결과 생성이 “치명적 리스크 중심/포지션 우선/중복 금지”로 동작하도록 출력 정책과 검증을 추가한다.

## 구현 Tasks
- T1. 포지션 판정 강화
  - 계약서 도입부/정의 조항에서 우리 측 역할(공급자/수요자, 도급인/수급인 등)을 추정하고 결과에 노출한다.
  - 퍼시스 기준 기본 규칙을 우선 적용한다(대리점=공급업자(갑), 인테리어/공사=수급인(을)).
- T2. 치명 리스크 Top 3 선정 로직 도입
  - 금전/권리/규제 카테고리별 후보를 만들고, 중복 제거 후 Top 3~5만 요약한다.
  - “High Risk” 표기는 최대 5개로 제한한다.
- T3. 조항별 목적 준수(토픽 오염 가드레일) 강화
  - 목적/정의 등 핵심 아닌 조항에 비용/정산 등 무관 문구가 들어가는 케이스를 차단한다.
- T4. 중복 금지(코멘트 dedup) 강화
  - 동일/유사 코멘트는 가장 적절한 핵심 조항 1곳에만 남기고, 나머지는 참조/링크로 축약한다.
- T5. 비즈니스적 대안 템플릿 적용
  - “협의로 수정/서면 통지/상한(캡)/증빙/이의제기/귀책 기준” 등 실무적 수정안 패턴을 출력에 반영한다.
- T6. 출력 포맷 고정 및 검증
  - [1]/[2]/[3] 섹션이 누락되면 실패 처리 또는 재생성한다.
  - [2] Top 3 항목은 “조항/리스크(1문장)/수정 제안(구체 문구)” 3요소가 모두 있어야 한다.
- T7. 회귀 테스트 추가
  - 대표 계약(대리점/공사/물품공급 등)에 대해 Top 3와 중복 금지, 토픽 오염 금지를 검증한다.
