# 167. rewrite 엔진 “최소 수정 원칙” 적용(앱 개발계약 우선)

## 문제
- 기존 suggested_rewrite는 원문 뒤에 긴 보강문을 괄호로 덧붙이는 방식이 많아
  - Word diff가 거의 전체 문장이 바뀐 것처럼 보이고,
  - 과잉수정 인상이 강하며 가독성이 떨어졌다.

## 목표
1) 최소 수정 원칙으로 변경(문장 직접 수정 우선)  
2) 이미 적절한 문구는 유지  
3) 문제되는 표현만 교체/삽입  
4) 긴 보강문 덧붙임(괄호 뒤첨) 지양  
5) app dev 핵심 쟁점은 기존 조항에 “흡수” 방식 우선  
6) clause-specific rewrite 품질 개선  

## 변경 사항(핵심)
- 파일: [rewrite_engine.py](file:///c:/Users/FURSYS/Desktop/aouribot/aouri-bot/runtime/review/rewrite_engine.py)
- 앱 개발계약(APP-001/002/003/004/006/007/009/010/011) rewrite 로직을 아래 방식으로 교체:
  - `별도 협의/추후 협의/상호 협의/TBD` 등 **애매 문구가 포함된 라인**을 우선 찾아 **해당 라인을 최소 문장으로 교체**
  - 애매 문구 라인이 없고 토픽만 존재하면, **해당 토픽 라인 바로 다음에 1문장 정도를 삽입**
  - 조항 전체 뒤에 긴 문단을 붙이는 방식은 사용하지 않음

## 적용된 최소 수정 패턴 예시
- IP 귀속(APP-001): “귀속은 별도 협의” → “산출물(IP)은 갑(또는 당사)에 귀속…” 문장으로 라인 교체
- 오픈소스(APP-002): “오픈소스는 추후 협의” → “SBOM/카피레프트 사전승인/위반 시 시정·배상” 문장으로 라인 교체
- SOW(APP-003): “범위 변경은 협의” → “변경요청은 갑 서면 승인 + 승인 없는 추가비용 불인정” 라인 교체
- 검수(APP-004): “기준/기간은 별도 정함” → “서면 확정 + 서면 합격 전 간주검수 금지 + 보완/재검수” 문장 삽입
- SLA(APP-006): “SLA는 별도 정함” → “응답·복구/구제수단(감액·크레딧·해지) 포함 서면 확정” 문장 삽입
- 재위탁(APP-009): “재위탁 가능” → “사전 서면 승인 + 동일 책임” 라인 교체
- 데이터 반환/삭제(APP-010): “별도 협의” → “반환→삭제(백업/로그 포함)+증적” 문장 삽입
- 종료/인수인계(APP-011): “별도 협의” → “자료 제공+전환 협력” 문장 삽입

## 기대 효과
- Word redline에서 “바뀐 부분”만 집중적으로 표시되어 가독성이 개선된다.
- 조항별 수정이 과잉 보강문 형태가 아니라 기존 문장에 흡수되어 실무적으로 자연스럽다.
